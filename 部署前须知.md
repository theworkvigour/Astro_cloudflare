# 部署前须知

## 概述

本项目基于 Astro + Cloudflare Workers 架构，支持**多语言独立部署**。
每种语言是一个完全独立的 Cloudflare Workers 站点，部署在独立的子域名上。

---

## 一、Cloudflare 一次性准备工作

以下操作只需在首次部署前执行一次，后续部署由 CI/CD 全自动完成。

### 1. 获取 Cloudflare Account ID

| 步骤 | 操作 |
|---|---|
| 1 | 登录 [Cloudflare Dashboard](https://dash.cloudflare.com) |
| 2 | 在右侧 **Account ID** 处点击复制按钮 |

![Cloudflare Account ID 位置](https://developers.cloudflare.com/cloudflare-one/static/documentation/account-id.png)

### 2. 创建 Cloudflare API Token

| 步骤 | 操作 |
|---|---|
| 1 | Cloudflare Dashboard → **My Profile**（右上角头像）→ **API Tokens** |
| 2 | 点击 **Create Token** |
| 3 | 选择 **Create Custom Token** |
| 4 | 设置以下权限： |

| 权限 | 值 |
|---|---|
| Account → Workers Scripts → **Edit** |
| Account → Workers KV Storage → **Edit** |
| Zone → DNS → **Edit** | 仅当使用自定义域名时需要 |
| Zone → Workers Routes → **Edit** | 仅当使用自定义域名时需要 |

| 步骤 | 操作 |
|---|---|
| 5 | 设置 **TTL** 为 Never |
| 6 | 点击 **Create** 并**立即复制 Token**（关闭后不再显示） |

### 3. 设置 GitHub Secrets

将上面获取的值填入 GitHub 仓库：

| 步骤 | 操作 |
|---|---|
| 1 | 打开 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions** |
| 2 | 点击 **New repository secret** |
| 3 | 添加以下 3 个 Secret： |

| Secret 名称 | 值 |
|---|---|
| `CLOUDFLARE_ACCOUNT_ID` | 从 Cloudflare Dashboard 复制的 Account ID |
| `CLOUDFLARE_API_TOKEN` | 上一步创建的 API Token |
| `SESSION_SECRET` | 任意安全的随机字符串（可用 `openssl rand -hex 32` 生成） |

> **注意：** `CLOUDFLARE_API_TOKEN` 权限不足会导致部署失败。必须包含 Workers Scripts 和 Workers KV Storage 的 Edit 权限。

## 二、可选设置

### 自定义域名

如果希望绑定域名（如 `en.alluredna.com`、`fr.alluredna.com`），在 GitHub 仓库添加变量：

| 位置 | 变量名 | 值 |
|---|---|---|
| GitHub → Settings → Secrets and variables → Actions → **Variables** | `CUSTOM_DOMAIN` | `true` |

映射规则：`{locale}.alluredna.com` → `vectoflare-{locale}` Worker

### Cloudflare KV Namespaces

CI/CD 会自动查找并绑定已存在的 KV Namespace。如需手动创建：

```bash
npx wrangler kv namespace create vectoflare-en-session
npx wrangler kv namespace create vectoflare-fr-session
# 每个启用的语言各执行一次
```

## 三、语言管理

### 启用/禁用语言

1. 登录后台管理界面 `/keystatic`
2. 点击 **Languages** 标签
3. 使用开关切换语言的启用/禁用状态
4. 点击 **Save & Push**

提交后自动触发 CI/CD 流水线，只会构建和部署**已启用**的语言。

> **English 始终启用不可禁用。**

### 添加新语言

1. 在 **Languages** 页面点击 **+ Add Language**
2. 输入语言代码（如 `ja`）、显示名称（如 `日本語`）、文字方向
3. 系统自动创建配置文件、导航文件和页面内容副本
4. 新语言默认**禁用**，需手动启用后才部署

### 删除语言

在 **Languages** 页面点击语言旁的 **Delete** 即可移除。

## 四、部署流程

### 自动部署（推荐）

每次向 `main` 分支推送代码时，CI/CD 自动：

1. 运行代码检查（astro check + ESLint + Prettier）
2. 读取 `src/data/site/languages.yaml` 获取已启用的语言列表
3. 依次构建每个已启用的语言版本
4. 部署到对应的 Cloudflare Workers
5. 设置 `SESSION_SECRET` 环境变量
6. 映射自定义域名（如已配置 `CUSTOM_DOMAIN=true`）

### 手动部署

```bash
# 构建指定语言
SITE_LOCALE=fr npm run build

# 部署到 Cloudflare Workers
cd dist/server
npx wrangler deploy --config wrangler.ci.json --name vectoflare-fr

# 设置 Session Secret
echo "your-secret" | npx wrangler secret put SESSION_SECRET --name vectoflare-fr
```

### 本地测试（构建所有已启用语言）

```bash
npm run build:all
```

## 六、注意事项

1. **`SESSION_SECRET` 必须设置**，否则联系表单加密功能会使用不安全的 fallback
2. 语言切换在 `.github/workflows/actions.yaml` 中通过 `enabled-locales` 任务动态读取
3. 在 Languages 页面修改启用状态后，需要等待 CI/CD 完成部署才能生效（通常 1-3 分钟）
4. 阿拉伯语（ar）使用 RTL（右到左）文字方向
5. `CLOUDFLARE_API_TOKEN` 不要使用过于宽泛的权限，建议按上述表格仅授权必要权限
